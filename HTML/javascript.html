let y const

let: te permite declarar variables limitando su alcance (scope) al bloque, declaración, o expresión donde se está usando.a diferencia de la palabra clave var la cual define una variable global o local en una función sin importar el ámbito del bloque. La otra diferencia entre var y let es que este último se inicializa a un valor sólo cuando un analizador lo evalúa (ver abajo). con let no se puede declarar con el mis nombre la variable

const: en esta ocacion no se puede cambiar la variable ejemplo si es un aarray no puede cambiar mas adelante por un numero o un string se puede modicar lo que hay dentro del array

destructuring

esto nos permite destructurar la infomacion de un objete y no estar declarando variables y gastanso linias de codigo ejemplo en el archivo destructuring.js. para destructurar un array la condicion es que el mismo tiene pocos elemtos y tenemos claro que elemento tiene

Spread & Rest

Ahora vamos a conocer 2 operadores introducidos en ES6 que llegaron a cambiar la forma de escribir código. En realidad, podríamos considerarlo como un solo operador ya que en los 2 casos se representa con tres puntos suspensivos (...), pero dependiendo de su posición y contexto cambia su función. los ejemplos para que quede mas claro estan en el archivo de spread&reset.js

En conclusión...

...la exploración de las características fundamentales de ECMAScript durante esta clase ha resaltado la importancia del estándar en el desarrollo de aplicaciones web modernas no sólo al ampliar nuestras habilidades como desarrolladores, sino que también contribuye a la creación de código más limpio, legible y eficiente.

  • Operadores Rest y Spread: Estos operadores permiten manejar un número variable de argumentos en funciones (Rest) y copiar elementos de arrays u objetos de manera más concisa (Spread), mejorando la flexibilidad y legibilidad del código.
  • Variables let y const: Introducidas en ECMAScript 6, las variables let y const ofrecen alternativas a var para declarar variables, brindando un mejor alcance y manejo de asignación constante, respectivamente.
  • Destructuring: Esta característica permite descomponer arrays y objetos en variables más pequeñas, simplificando la asignación y extracción de valores de estructuras complejas.
  • Template Objects y Template Literals: Los Template Literals facilitan la creación de cadenas de texto multilínea y la interpolación de variables de manera más limpia. Los Template Objects extienden esta funcionalidad permitiendo la creación de objetos de manera similar.
